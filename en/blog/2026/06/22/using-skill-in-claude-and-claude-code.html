🧩 Using Skills in Claude Code and Claude

June 22, 2026 · 5 min read

What is a skill, and how do you install and use one in Claude and Claude Code?

What is a skill​

Put simply, a skill is just a folder containing at least one SKILL.md file that spells out "when to use this capability" and "exactly how to do it." The example I'll use here is a skill I had Claude write with skill-creator, called blog-markdown-formatter. Next up: how to install it and how to use it.

Personally it reminds me a bit of a mod for a single-player game — except this "mod" exists purely to make the experience better.

Installing in Claude Code​

Step 1: Download the .skill file​

If your skill was generated by Claude Code's skill creator, or made some other way — doesn't matter the source, as long as it came from Claude — the file should look like this:

An orange "SKILL" icon, and in Finder there's even an "Open with Claude" button — that's because the Claude desktop app installed on your machine registers a custom style for the .skill extension. It doesn't mean the file format itself has changed. Don't worry, underneath it's still a standard zip archive — the extension and icon are purely cosmetic.

If you still don't believe it, open a terminal and run:

file ~/Downloads/blog-markdown-formatter.skill

✅ Normal: the output mentions Zip archive data, confirming it's really a zip

❌ Abnormal: if the output says something else, the download got corrupted — try downloading again

Step 2: Unzip It​

cd ~/Downloads
unzip blog-markdown-formatter.skill -d ~/.claude/skills/

Step 3: verify the directory structure​

ls ~/.claude/skills/blog-markdown-formatter

✅ Normal: you see SKILL.md and references/ sitting directly inside this folder

❌ Abnormal: if what you see is a .skill file sitting inside (meaning you just moved the zip, didn't unzip it), go back to Step 2 and run unzip again

Once unzipped correctly, the directory structure should look like this:

~/.claude/skills/blog-markdown-formatter/
├── SKILL.md
└── references/
    ├── blog-style-guide.md
    └── syntax-checklist.md

At this point, installing the skill in Claude Code is done.

Installing in Claude​

  1. In the web app or desktop app, go to Settings → Customize → Skills
  2. Click "+" → "Create skill" → "Upload a skill"
  3. Upload the downloaded .skill file directly (no need to unzip — the web app parses it for you)
  4. Once uploaded, toggle it on in the list

At this point, installing the skill in Claude is done.

Note: installing a skill on Claude doesn't mean Claude Code can use it too — you install separately on each side, they're not the same thing.

Updating a Skill​

After a skill's content changes (rule tweaks, new features), it doesn't automatically sync to the version you already installed locally — you need to manually overwrite both sides.

On Claude Code:

cd ~/Downloads
rm -rf ~/.claude/skills/blog-markdown-formatter
unzip blog-markdown-formatter.skill -d ~/.claude/skills/

On Claude: go to Settings → Customize → Skills, find the skill, delete the old one, and upload the new file.

Skill Usage​

Triggering a skill is simple. Take this skill as an example: its SKILL.md has a description along these lines — "when the user says things like 'help me clean up the formatting on this markdown,' 'tidy up this document's formatting,' 'make this blog post nicer,' 'this post's formatting is a mess, help me fix it,' 'reformat/rewrite this in my blog's style' ..."

That description lives in the description field of SKILL.md. If you want to know exactly when a skill gets triggered, just open it and take a look.

Automatic triggering​

Just tell Claude or Claude Code directly:

Help me clean up the markdown formatting on /document/strawberry.txt, in my blog's style

Claude or Claude Code will decide on its own, based on the skill's description, whether to invoke it.

To check whether it actually used the skill, look at its thinking process — it'll show whether it went and read SKILL.md.

Manual Invocation​

A .claude/skills/<name>/SKILL.md directory automatically gets a matching slash command. For this skill, that's:

/blog-markdown-formatter /document/strawberry.txt

You don't need to write out a full prompt, but the slash command needs at least one target after it (like a file path) — you can't leave it empty:

  • With something after it: if SKILL.md doesn't define an $ARGUMENTS placeholder, Claude Code automatically appends what you typed as ARGUMENTS: <what you typed>, tacked onto the end of the skill's rules, so Claude knows exactly what to work on
  • With nothing after it: only the formatting rules get loaded into context, with no idea what file to act on — it'll probably just ask you

Summary​

Not sure there's much to sum up — if you've read this far, you already know how to use a skill.

So, congratulations — you've been congratulated.
