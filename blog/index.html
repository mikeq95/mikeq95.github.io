🌐 macOS 网络重置命令速查

May 16, 2026 · One min read

一般情况下，只需运行 DNS 刷新 + DHCP 重置 即可解决大多数网络问题。

常用命令（按推荐顺序）​

1. 刷新 DNS 缓存​

sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder

2. 释放并重新获取 IP（DHCP）​

sudo ipconfig set en0 DHCP

3. 重启 Wi-Fi​

networksetup -setairportpower en0 off && networksetup -setairportpower en0 on

4. 重启网络接口​

sudo ifconfig en0 down && sudo ifconfig en0 up

🔧 完整重置网络设置（较彻底）​

删除网络偏好设置文件，重启后自动重建：

sudo rm /Library/Preferences/SystemConfiguration/com.apple.network.identification.plist
sudo rm /Library/Preferences/SystemConfiguration/NetworkInterfaces.plist
sudo rm /Library/Preferences/SystemConfiguration/preferences.plist

⚠️ 注意： 执行后需重启 Mac，所有网络配置将重置为默认状态，Wi-Fi 等需重新配置。

📌 补充说明​

接口说明
en0通常为 Wi-Fi 无线网卡
en1通常为有线以太网卡

如不确定接口名称，可运行 ifconfig 查看确认。

搜索引擎使用指南

May 16, 2026 · 2 min read

一、搜索引擎选择​

推荐排序​

抛弃百度，以下引擎均可替代：

引擎适合场景
Google日常搜索首选
DuckDuckGo争议话题、隐私保护
Yandex劲爆内容、软件资源、音乐
Brave日常搜索备选
Bing（国内版）萌新过渡使用

各引擎特点​

  • Google：搜争议内容时结果偏 PC，反方观点少且力度弱，不够尖锐
  • DuckDuckGo：比 Google 中立，搜索词无高亮显示（可用脚本改善，见文末）
  • Yandex：完全不理会西方政治正确那套，适合搜敏感/劲爆内容
    • 例：搜「爱泼斯坦名单」，只有 Yandex 将名单置顶
    • 例：搜「好莱坞丑闻」，只有 Yandex 显示激烈结果
  • 政治正确程度：Google > Brave > DuckDuckGo > Yandex

二、实用搜索技巧​

软件资源（学生党）​

用 Yandex 搜索：

软件名 + cracked / premium / unlocked

安卓、PC、Mac 均适用。

免费音乐下载（学生党）​

用 Yandex 搜索：

歌名 + free download
歌名 + Скачать

结果中找带俄语的链接，一般可直接下载。

限定网站搜索​

关键词 site:https://x.com

适合某些自带搜索功能较差的资源网站。

搜电子书（Anna's Archive / Z-Library）​

关键词 + edition      # 搜再版书，质量通常更好，例：tort edition
关键词 + 译           # 搜中译版，例：文明史 译

用豆瓣找好书​

豆瓣搜索关键词 → 筛选高分书单；书籍详情页下方也会推荐同类书籍。

三、DuckDuckGo 搜索词高亮脚本​

需安装 Tampermonkey 浏览器插件后使用。

javascript

// @name         DuckDuckGo 搜索词高亮

// @description  为 DuckDuckGo 搜索结果中的关键词添加高亮颜色
// @author       You
// @match        https://duckduckgo.com/?*
// @grant        GM_addStyle
// ==/UserScript==
(function() {
    'use strict';
    GM_addStyle(`
        .kY2IgmnCmOGjharHErah span b {
            color: #ff897e !important;
        }
    `);
})();

用 Kokoro 把文章变成播客音频 —— 从零部署教程

May 16, 2026 · 4 min read

本文记录了如何在 Mac (Apple Silicon) 上通过 Docker 本地部署 Kokoro TTS，将英文/中文文章转换为高质量音频。

一、为什么选 Kokoro？​

在调研了多个开源 TTS 方案后，最终选择 Kokoro + kokoro-web 的原因：

方案Stars英文效果声音种类协议
Kokoro7k+⭐ 顶级88 种可选Apache 2.0
ChatTTS~33k中等随机生成，不可选CC BY-NC（不可商用）
CosyVoice高一般中文最强Apache 2.0

Kokoro 的核心优势：

  • 英文效果接近 ElevenLabs 商业级别
  • 88 种声音（美式/英式，男/女，A~D 分级）
  • 82M 参数，轻量，M1 Mac CPU 可直接运行
  • Apache 2.0，免费商用

二、声音分级说明​

Kokoro 的声音按质量分为 A~D 级，A 级最高：

English (US) 推荐​

声音级别风格
HeartA温暖女声（推荐首选）
BellaA-高质量女声
NicoleB-自然女声
MichaelC+成熟男声

English (UK) 推荐​

声音级别风格
EmmaB-英式女声
GeorgeC英式男声

💡 每个声音第一次使用时会下载对应的 .bin 文件，之后自动缓存，无需重复下载。

三、部署方式对比​

Docker Compose源码运行纯浏览器（WebGPU）
难度⭐⭐⭐⭐⭐⭐
需要安装DockerNode.js + npm无
推荐场景本地长期使用开发者改源码偶尔体验

结论：日常使用推荐 Docker Compose 方式。

四、环境准备​

1. 安装 Docker Desktop​

由于国内访问 Docker 官网较慢，建议从阿里云镜像下载：

https://mirrors.aliyun.com/docker-toolbox/mac/docker-for-mac/

下载 .dmg 文件，双击安装，打开 Docker Desktop 等待启动（菜单栏出现 🐳 图标）。

2. 配置命令行路径（永久生效）​

echo 'export PATH="$PATH:/Applications/Docker.app/Contents/Resources/bin"' >> ~/.zshrc
source ~/.zshrc

3. 验证安装​

docker --version && docker compose version

输出类似以下内容即为成功：

Docker version 29.4.3, build 055a478
Docker Compose version v5.1.3

五、部署 kokoro-web​

1. 创建项目目录​

mkdir kokoro-web && cd kokoro-web

2. 创建 compose.yaml​

cat > compose.yaml << 'EOF'
services:
  kokoro-web:
    image: ghcr.io/eduardolat/kokoro-web:latest
    ports:
      - "3000:3000"
    environment:
      - KW_SECRET_API_KEY=my-secret-key
    volumes:
      - ./kokoro-cache:/kokoro/cache  # 模型缓存，避免重复下载
    restart: unless-stopped
EOF

KW_SECRET_API_KEY 是本地服务的访问密码，本地使用随便填即可。

3. 启动服务​

docker compose up -d

第一次启动会拉取镜像，需要几分钟。国内访问 ghcr.io 较慢，耐心等待。

4. 验证运行​

docker compose logs -f

看到 Listening on http://0.0.0.0:3000 即为成功。

六、使用 WebUI​

打开浏览器访问：

http://localhost:3000

关键设置​

选项推荐值说明
Execution placeAPI (Self-hosted)使用本地 Docker 模型，而非浏览器
API URLhttp://localhost:3000/api/v1本地服务地址
API Keymy-secret-key就是 compose.yaml 里设置的那个
Model quantizationmodel (q8f16)速度和音质最平衡
Voice qualityHeart (A)推荐首选

⚠️ 重要：Execution place 一定要选 API (Self-hosted)，否则会在浏览器内运行，速度很慢。

七、关于 Markdown 文件输入​

Kokoro 原生只接受纯文本，直接输入 .md 文件会把 ##、**bold** 等 Markdown 语法也读出来。

解决方案：用 pandoc 先转换：

# 安装 pandoc
brew install pandoc

# 转换并生成音频
pandoc article.md -t plain | kokoro-tts - output.mp3 --voice am_michael

八、生成速度说明​

影响生成速度的因素：

  • 文章长度：线性增长，300 词约需 30~60 秒
  • 模型量化：fp32 最慢，q8f16 平衡，q4 最快
  • 运行位置：API (Self-hosted) 用 CPU 跑；Browser 用 GPU 跑但首次需下载模型到浏览器

九、常用管理命令​

docker compose logs -f    # 查看实时日志
docker compose stop       # 停止服务
docker compose start      # 启动服务
docker compose restart    # 重启服务
docker compose pull       # 更新到最新版本

问题-中国人在国外会被种族歧视嘛？

May 14, 2026 · 2 min read

首先说明，我没有出过国，但最近密集的网络冲浪让我对国外有了一些了解，在这里分享一下，也顺带回答这个问题。

我目前主要了解的是英国——因为有个英国朋友，所以一直关注这个国家。

直接回答：会。

但英国的种族歧视是一种隐性歧视，不容易被察觉，却非常普遍。我从媒体角度来解释这个现象。

长期以来，英国小报对中国的负面报道，培养出了一批"读过报纸的歧视者"。他们不会当面骂你，但会突然抛出一句："你们中国人是不是都吃狗肉？"——用"好奇"的语气来包装冒犯。你认真解释之后，他们会说"哦，原来如此"，然后下次再换一个刻板印象继续问。本质上，他们并不是真的想了解你，只是在用你来验证从媒体里得来的那套认知。

这和真正的好奇心有本质区别。举个反例：你作为中国人，问英国人"你们为什么喜欢把奶和茶混在一起喝？"——这是单纯的疑惑，是对一种陌生文化习惯的好奇，而不是在套用任何刻板印象。

两者的区别在于：前者是带着结论去找验证，后者是带着疑问去寻求理解。

问题: 为什么不能找原生家庭不好的女生做女朋友?

May 14, 2026 · 2 min read

准确的来说，不能找那种原生家庭不好，身弱但又带一点姿色的女生。这种人一般烂花开满地，你以为是她人不行吗？让我告诉你真正的原因：

因为一个缺爱的人，她很难去爱上别人。这种人一般就是：谁在她身边，她就爱谁，如果说这个人离开了，她也会难过一阵子，但只要有新的人出现，她就是立马忘记旧的人，重新开始。

看上去，她好像爱上了很多人，其实她谁都不爱。她只是想有一个人陪着她，有人能听她的话，有人以她为中心，她在找那种缺失的关注和依赖。

这个不是爱，她因为极度的缺乏安全感，所以需要陪伴，因为极度缺爱，所以需要被爱。但她自己，根本就爱不上别人，她不认为自己能够真正的被别人爱。

所以，当你作为她的伴侣，你就能够时常感受到，她在不停索取爱，确认爱，让你证明爱。

当然，我要做一个补充，其实这个应该算作一个我的个人情感经历，这个文章作为我的反思，不是普遍规律，还有人原生家庭很好，照样感情混乱，或者有些原生家庭不好的人,经过自我成长，反而对感情格外珍视和忠诚，我只是说"部分人的特征"，不是这类人的普遍规律。

编程英语单词短语收集

April 29, 2026 · 3 min read

  1. demonstrate

    • 例句: This tutorial no longer ==demonstrates== current SwiftUI or Xcode practices.
    • 译文: 本教程不再演示当前的 SwiftUI 或 Xcode 实践。
    • 释义: 演示

  2. prevent

    • 例句: This option will ==prevent== Shadowrocket from being set as the default route.
    • 译文: 此选项将阻止 Shadowrocket 被设置为默认路由。
    • 释义: 阻止

  3. modifier

    • 例句: A new ==modifier== creates a new view based on the original one.
    • 译文: 一个新的修饰符基于原始的样子，会创建一个新的视图。
    • 释义: 修饰符

  4. components

    • 例句: One of the most common and recognizable user interface ==components== is the combination of an icon and a label.
    • 译文: 最常见、最容易识别的用户界面组件之一，就是图标+文字的组合。
    • 释义: 零件，组件

  5. idiom

    • 例句: This ==idiom== appears across many kinds of apps and shows up in collections, lists, menus of action items, and disclosable lists, just to name a few.
    • 译文: 这种惯用组合出现在许多类型的应用程序中，也出现在集合视图、列表、操作菜单以及可展开列表里——仅举几例而已。
    • 释义: 惯用模式，惯用组合

  6. just to name a few

    • 例句: This idiom appears across many kinds of apps and shows up in collections, lists, menus of action items, and disclosable lists, ==just to name a few==.
    • 译文: 这种惯用组合出现在许多类型的应用程序中，也出现在集合视图、列表、操作菜单以及可展开列表里——仅举几例而已。
    • 释义: 仅举几例 / 只是列举这么多

  7. interface

    • 例句: You create the user ==interface==, or UI, for your app inside the body, the entire contents of your app are in the code below.
    • 译文: 你在 body 属性中创建应用的用户界面（UI），应用程序的全部内容都在下面的代码中。
    • 释义: 界面
// NOTE: - 演示代码
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundColor(.tint)
            
            Text("Hello World")
        }
    }
}

#Preview {
    ContentView()
}

  1. instance

    • 例句: To construct your narrative, the app needs an ==instance== of Story, which takes an array of StoryPage instances.
    • 译文: 要构建你的叙事，该应用需要一个 Story 实例，该实例接受一个 StoryPage 实例数组。
    • 释义: 实例

  2. hold

    • 例句: The text property ==holds== a String that describes the choice readers can make.
    • 译文: 文本属性保存一个字符串，用于描述读者可以做出的选择。
    • 释义: 保存，持有

  3. present

    • 例句: Use a navigation stack to ==present= a stack of views over a root view.
    • 译文: 使用导航堆栈在根视图上呈现一系列视图。
    • 释义: 显示，呈现

  4. indicates

    • 例句: A value-destination ==indicates== that you are pushing a value onto the path.
    • 译文: "Value destination"这种用法表明/意味着你正在将一个值推送到导航路径（Navigation path）上。
    • 释义: 表明，意味着

通用英语单词短语收集

April 29, 2026 · 2 min read

  1. Based on the principle of voluntariness

    • 例句: Membership in the union is ==based on the principle of voluntariness==—no employee is forced to join.
    • 译文: 加入工会遵循自愿原则，没有任何员工被强迫参加。
    • 释义: 根据自愿的原则

  2. One for all, all for one

    • 例句: The three friends made a pact: ==one for all, all for one==. They promised to stand by each other no matter what challenges lay ahead.
    • 译文: 三位朋友立下誓言：人人为我，我为人人。他们承诺无论未来面临何种挑战，都会相互扶持。
    • 释义: 人人为我，我为人人

  3. Be careful, though

    • 例句: The restaurant is amazing and cheap. ==Be careful==, though: it gets really crowded on weekends.
    • 译文: 那家餐厅很棒，还很便宜。不过要注意：周末人会特别多。
    • 释义: 不过要注意（用于补充警示信息）

  4. embedded in

    • 例句: These attitudes are ==embedded in== our society.
    • 译文: 这些观念根植于我们的社会之中。
    • 释义: 根植于，深深嵌入

  5. enthusiasts

    • 例句: She is one of the most dedicated fitness ==enthusiasts== I have ever met.
    • 译文: 她是我见过的最投入的健身爱好者之一。
    • 释义: 爱好者，发烧友
